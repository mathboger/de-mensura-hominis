# Setup guide

This guide walks you through everything needed to build the book locally.
It is written for **WSL2 (Ubuntu)** on Windows. All commands run in your
WSL2 terminal unless stated otherwise.

The guide is divided into two parts:

- **Part 1 — System configuration** — things you install once on your
  machine. Skip any step you have already completed.
- **Part 2 — Project setup** — things tied to this specific repository
  that must be done every time you clone it fresh on any machine.

---

## Part 1 — System configuration

---

### 1.1 Open a WSL2 terminal

Press `Win + S`, search **Ubuntu** (or your WSL distro), open it.
Everything below runs here.

---

### 1.2 Core system packages

Check whether the essentials are present:

```bash
which make git curl wget python3 R cmake
```

Install anything missing:

```bash
sudo apt update && sudo apt install -y \
  build-essential \
  make \
  git \
  curl \
  wget \
  python3 \
  python3-pip \
  python3-venv \
  software-properties-common \
  dirmngr \
  fontconfig \
  libpng-dev \
  libxml2-dev \
  pandoc \
  cmake
```

> `libpng-dev` and `libxml2-dev` are required to compile the R packages
> `reticulate` and `xml2`. `cmake` is required by the R package `fs`.
> Installing all three here avoids cryptic compilation errors later.

---

### 1.3 Quarto

Check if Quarto is installed:

```bash
quarto --version
```

If not, install the Linux build (not the Windows installer):

```bash
wget "https://github.com/quarto-dev/quarto-cli/releases/latest/download/quarto-linux-amd64.deb"
sudo dpkg -i quarto-linux-amd64.deb
rm quarto-linux-amd64.deb
quarto --version
```

---

### 1.4 TinyTeX (for PDF output)

Check if TinyTeX is installed:

```bash
which tlmgr
```

If not:

```bash
quarto install tinytex
```

Add TinyTeX to your PATH permanently — this must be done before running
`tlmgr`, otherwise the commands below will not be found:

```bash
echo 'export PATH="$HOME/.TinyTeX/bin/x86_64-linux:$PATH"' >> ~/.bashrc
source ~/.bashrc
which tlmgr   # should now return a path
```

Force-reinstall the core bundles. TinyTeX sometimes marks packages as
present without actually unpacking their files — this ensures they are
physically on disk:

```bash
tlmgr install --reinstall tools koma-script latex-tools-dev
```

Install the remaining packages needed by this book:

```bash
tlmgr install \
  fontspec microtype tcolorbox caption booktabs \
  etoolbox tex-gyre inconsolata fvextra upquote \
  titlesec scrlayer-scrpage
```

Install the book's fonts as system fonts so XeLaTeX can find them by
name (TinyTeX's internal font tree is not always visible to XeLaTeX's
font lookup):

```bash
sudo apt install -y fonts-texgyre fonts-inconsolata
fc-cache -fv
```

Verify both fonts are visible:

```bash
fc-list | grep -i pagella
fc-list | grep -i inconsolata
```

Both commands should return at least one path. If either returns nothing
the PDF build will fail with a font-not-found error.

---

### 1.5 R

Check the installed version:

```bash
R --version
```

If R is not installed or is older than 4.0, install from CRAN's Ubuntu
repository (the version in Ubuntu's default `apt` repo is outdated):

```bash
wget -qO- https://cloud.r-project.org/bin/linux/ubuntu/mariner.gpg \
  | sudo tee /etc/apt/trusted.gpg.d/cran.asc > /dev/null

. /etc/os-release
sudo add-apt-repository \
  "deb https://cloud.r-project.org/bin/linux/ubuntu ${VERSION_CODENAME}-cran40/"

sudo apt update && sudo apt install -y r-base r-base-dev
R --version   # should print 4.x
```

### 1.6 VS Code (recommended editor)

If you use VS Code, install the **WSL** extension in Windows, then open
the project from the WSL terminal:

```bash
code .
```

Recommended extensions to install inside the WSL context:

- `quarto.quarto` — live preview, syntax highlighting
- `ms-python.python` — Python support
- `REditorSupport.r` — R support
- `ms-toolsai.jupyter` — Jupyter notebook support

For R to attach properly in VS Code, also install:

```bash
pip install radian
R -e "install.packages('languageserver', repos='https://cloud.r-project.org')"
R -e "install.packages('httpgd', repos='https://cloud.r-project.org')"
```

Then add to VS Code's settings JSON (`Ctrl+Shift+P` → Open User Settings JSON):

```json
{
  "r.rpath.linux": "/usr/bin/R",
  "r.rterm.linux": "/home/YOUR_USERNAME/.local/bin/radian",
  "r.bracketedPaste": true,
  "r.plot.useHttpgd": true
}
```

### 1.7 Optional — match PDF fonts to HTML

By default the PDF uses TeX Gyre Pagella (body) and Inconsolata (code),
which are classic book fonts that require no extra installation. If you
want the PDF to use the same fonts as the HTML output (Jost, Outfit,
JetBrains Mono), follow these steps.

Download and install the font files directly from the Google Fonts
GitHub repository (the Google Fonts website download URL requires a
browser and does not work with wget):

```bash
mkdir -p ~/.local/share/fonts

# Jost (variable font — contains all weights)
wget -q -O ~/.local/share/fonts/Jost-Regular.ttf \
  "https://github.com/google/fonts/raw/main/ofl/jost/Jost%5Bwght%5D.ttf"
wget -q -O ~/.local/share/fonts/Jost-Italic.ttf \
  "https://github.com/google/fonts/raw/main/ofl/jost/Jost%5Bital%2Cwght%5D.ttf"

# Outfit (variable font)
wget -q -O ~/.local/share/fonts/Outfit-Regular.ttf \
  "https://github.com/google/fonts/raw/main/ofl/outfit/Outfit%5Bwght%5D.ttf"

# JetBrains Mono (variable font)
wget -q -O ~/.local/share/fonts/JetBrainsMono-Regular.ttf \
  "https://github.com/google/fonts/raw/main/ofl/jetbrainsmono/JetBrainsMono%5Bwght%5D.ttf"
wget -q -O ~/.local/share/fonts/JetBrainsMono-Italic.ttf \
  "https://github.com/google/fonts/raw/main/ofl/jetbrainsmono/JetBrainsMono%5Bital%2Cwght%5D.ttf"

fc-cache -fv
fc-list | grep -i jost
fc-list | grep -i outfit
fc-list | grep -i "JetBrains"
```

All three should return paths. Then open `build/preamble.tex`, comment
out the Default fonts block, and uncomment the Optional block as
instructed in the comments inside that file.

## Part 2 — Project setup

Run these steps every time you clone this repository fresh on any machine.

---

### 2.1 Clone into the WSL filesystem

Keep the project inside WSL — not on the Windows drive. File I/O across
the WSL boundary (`/mnt/c/...`) is slow enough to affect build times.

```bash
mkdir -p ~/projects
cd ~/projects
git clone https://github.com/yourusername/de-mensura-hominis.git
cd de-mensura-hominis
```

---

### 2.2 Global Python environment

Create the global virtual environment and install dependencies:

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install --upgrade pip
pip install -e ".[dev]"
```

Register the global Jupyter kernel so Quarto can find it:

```bash
python -m ipykernel install --user --name book --display-name "Book (Python)"
```

Optional — auto-activate the venv whenever you enter the project folder:

```bash
echo 'cd() { builtin cd "$@" && [ -f .venv/bin/activate ] && source .venv/bin/activate; }' \
  >> ~/.bashrc
source ~/.bashrc
```

---

### 2.3 Bootstrap chapter environments

Each chapter has its own Python virtual environment and Jupyter kernel.
These are not stored in git — they are reconstructed from the committed
`requirements.txt` files in each chapter folder. Run this once to
initialise all of them:

```bash
source .venv/bin/activate
make bootstrap
```

This is the Python equivalent of `renv::restore()` for R. Any time a
new chapter is added to the repository or a chapter's `requirements.txt`
changes, re-running `make bootstrap` brings all environments up to date.
To update a single chapter only:

```bash
make bootstrap-chapter CHAPTER=hodgkin-huxley
```

---

### 2.4 R environment

The repo ships with a `.Rprofile` that loads `renv` automatically on
startup. Because `renv` has not been initialised yet, R will crash if
run directly. Follow these steps in order.

**Step 1** — temporarily disable `.Rprofile`:

```bash
mv .Rprofile .Rprofile.bak
```

**Step 2** — open an R console and install required packages:

```bash
R
```

Inside the R console:

```r
install.packages(
  c('renv', 'yaml', 'reticulate', 'knitr', 'downlit', 'xml2'),
  repos = 'https://cloud.r-project.org'
)
renv::init()
renv::snapshot()
q()
```

**Step 3** — re-enable `.Rprofile` now that `renv/` exists:

```bash
mv .Rprofile.bak .Rprofile
```

> **What `renv.lock` is:** the R equivalent of `requirements.txt`. It
> records the exact version of every installed R package so the
> environment can be reproduced exactly with `renv::restore()`. Always
> commit `renv.lock` to git.
>
> **Adding R packages later:**
> ```r
> renv::install("ggplot2")   # install
> renv::snapshot()           # record in renv.lock
> ```

---

### 2.5 Download the citation style file

```bash
curl -o chicago-author-date.csl \
  https://raw.githubusercontent.com/citation-style-language/styles/master/chicago-author-date.csl
```

### 2.6 Optional — Playwright for app screenshots

Only needed if you use the `make screenshots` command to capture
Streamlit app screenshots for embedding in the PDF.

```bash
pip install playwright
playwright install chromium --with-deps
```

---

### 2.7 Preview the book

```bash
source .venv/bin/activate
make preview
```

Quarto starts a local server. Open the printed URL (e.g.
`http://localhost:4242`) in your **Windows browser** — WSL2 forwards the
port automatically. The page live-reloads as you save `.qmd` files.

---

### 2.8 Build

```bash
source .venv/bin/activate
make html     # HTML only — output lands in output/html/
make pdf      # PDF only  — output lands in output/pdf/
make all      # both
make fast     # both, skipping code re-execution (uses cached outputs)
```

## Daily writing workflow

```bash
cd ~/projects/de-mensura-hominis   # venv auto-activates if you set the bashrc hook

# ── Scaffold a new chapter ─────────────────────────────────────────────────────
# Creates chapters/<slug>/ with its own venv, registers the Jupyter kernel,
# and adds the chapter to _quarto.yml.
# Folder name is derived from the title — no numeric prefix is added.
# Chapter order is controlled by the chapters: list in _quarto.yml.

# Basic — adds to the flat chapter list
make chapter TITLE="The Hodgkin-Huxley Model"

# With a part assignment
make chapter TITLE="The Hodgkin-Huxley Model" PART="Book II: Neuron-Level Models"

# With a Streamlit demo app
make chapter-app TITLE="Bayesian Updating" PART="Book III: Individual Behaviour"

# ── Write and preview ──────────────────────────────────────────────────────────
# Edit chapters/<slug>/index.qmd in VS Code.
# Add chapter-specific packages to chapters/<slug>/requirements.txt, then:
cd chapters/the-hodgkin-huxley-model
source .venv/bin/activate
pip install -r requirements.txt
cd ../..

# Live preview while writing (hot-reload in browser)
make preview

# Render just one chapter for fast iteration
source .venv/bin/activate
python build/compile.py --html --chapter the-hodgkin-huxley-model

# ── Reorder chapters ───────────────────────────────────────────────────────────
# To change chapter order or reassign chapters to different parts,
# edit the chapters: list in _quarto.yml directly.
# No folder renaming is needed — folder names are independent of order.

# ── Before committing ──────────────────────────────────────────────────────────
make test     # verify all notebooks execute cleanly
make all      # full build
```

## Troubleshooting

**`make: command not found`**
```bash
sudo apt install make
```

**`tlmgr: command not found`**
```bash
echo 'export PATH="$HOME/.TinyTeX/bin/x86_64-linux:$PATH"' >> ~/.bashrc
source ~/.bashrc
```

**XeLaTeX cannot find a font**
```bash
sudo apt install fonts-texgyre fonts-inconsolata
fc-cache -fv
fc-list | grep -i pagella
fc-list | grep -i inconsolata
```

**R package fails to compile**
The most common cause is a missing system library. Check the error
message for a library name ending in `-dev` and install it:
```bash
sudo apt install libNAME-dev
```
Common ones: `libpng-dev`, `libssl-dev`, `libxml2-dev`,
`libcurl4-openssl-dev`, `cmake`.

**R crashes on startup with `cannot open file 'renv/activate.R'`**
The `.Rprofile` is trying to load renv before it has been initialised.
Follow section 2.4 exactly, starting with `mv .Rprofile .Rprofile.bak`.

**Jupyter kernel not found during build**
A chapter's kernel is not registered on this machine. Run:
```bash
source .venv/bin/activate
make bootstrap
```
Or for a single chapter:
```bash
make bootstrap-chapter CHAPTER=the-hodgkin-huxley-model
```

**Quarto can't find the R kernel**
```bash
R -e "install.packages('knitr', repos='https://cloud.r-project.org')"
```

**Port already in use on `make preview`**
```bash
quarto preview . --port 4243
```

**Output folders clash**
The build script outputs HTML to `output/html/` and PDF to `output/pdf/`
so they never overwrite each other. If you are seeing clashes, verify
that `build/compile.py` passes `--output-dir` correctly.

**Windows Zone Identifier files appearing**
These are harmless Windows metadata files created when downloading files
from the internet. Delete them with:
```bash
find . -name "*Zone.Identifier" -delete
```
They are already excluded from git via `.gitignore`.
